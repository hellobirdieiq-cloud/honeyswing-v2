# HoneySwing v1.3.0 — 120fps Capture + Cloud Video Upload

## Context

HoneySwing currently captures swings at the device's default frame rate (no `fps` or `format` prop on Camera), stores pose data to Supabase, and saves video locally only. This release adds two features: (1) upload swing videos to Supabase Storage for cloud backup, and (2) capture at 120fps when supported for smoother analysis input, with frame skipping to keep analysis at ~30fps.

```
User taps "Record Now" or "3-2-1"
  │
  ▼
startCountdownCapture() [line 273] or startInstantCapture() [line 295]
  │
  ▼
beginRecording() [line 249]
  ├─ Clears state (motion, analysis, refs)
  ├─ cameraRef.current.startRecording({onRecordingFinished, onRecordingError}) [line 257]
  │   └─ onRecordingFinished sets videoUriRef.current = video.path [line 259]
  │      then calls tryNavigate()
  ├─ Sets phase = 'capturing' [line 265]
  ├─ Plays go sound [line 266]
  └─ Sets 4s timeout → finalizeCapture() [line 268]

During capture (every frame):
  frameProcessor worklet [line 372]
  └─ honeyPoseDetect(frame) in worklet
  └─ appendPoseFrame() via Worklets.createRunOnJS [line 131]
      ├─ Updates skeleton overlay (every frame, all phases) [line 162]
      └─ If capturing: runs provider.detectFromFrame, pushes to motionFramesRef [line 169-181]

After 4s (or user taps stop):
  finalizeCapture() [line 196]
  ├─ clearTimers()
  ├─ cameraRef.current.stopRecording() [line 198]
  ├─ Quality gates (min frames, good frames) [lines 202-217]
  ├─ analyzePoseSequence(sequence) — synchronous [line 228]
  ├─ Stores to swingMotionStore [lines 230-235]
  ├─ Sets phase = 'complete' [line 236]
  ├─ persistSwing(frames, analysis, classification).catch(() => {}) — fire-and-forget [line 239]
  ├─ analysisReadyRef = true [line 241]
  ├─ Safety timeout 3s → force navigate with null video [line 242]
  └─ tryNavigate() [line 246]

tryNavigate() [line 113]
  ├─ Guards: phase=complete, analysisReady, videoUri !== undefined, not already navigated
  ├─ Sets video URI in swingMotionStore
  └─ router.push('/analysis/result')
```

**Key async boundary**: `stopRecording()` → native finalization → `onRecordingFinished` callback. The analysis + persist happen synchronously after `stopRecording()` but before the callback fires (native async). Safety timeout (3s) handles case where video never arrives.

**Race condition (existing)**: `tryNavigate()` called from both `finalizeCapture()` and `onRecordingFinished`. Protected by `navigatedRef`. No new race conditions introduced by this plan.

**persistSwing currently**: `supabase.from('swings').insert(row)` — returns void, does NOT return inserted ID (line 75).

## 5. Key Observations

1. **No format/fps configured** — Camera uses device defaults. No `format` or `fps` prop on `<ReanimatedCamera>`.
2. **No Supabase Storage usage anywhere** — bucket must be created in Supabase dashboard.
3. **persistSwing doesn't return ID** — needs `.select('id').single()` to get swingId for upload path.
4. **deleteAccount doesn't clean up storage** — only deletes swings + profiles rows. Must add storage cleanup.
5. **expo-file-system** is installed as transitive dep but not in package.json and not imported anywhere.
6. **useSharedValue** already imported from `react-native-reanimated` (line 7). Can reuse for frame counter.
7. **No videoCodec set** — `startRecording` currently passes only callbacks.
8. **PrivacyInfo.xcprivacy exists** — may need update for network data collection.

## 6. Implementation Plan

### Phase A: Cloud Video Upload (ship first, verify independently)

#### A1. Supabase prerequisite (manual, not code)

- Create storage bucket `swing-videos` in Supabase dashboard (private, no public access)
- Add columns to `swings` table: `video_storage_path text`, `video_uploaded_at timestamptz`
- Set RLS policy on bucket: authenticated users can upload to their own `<userId>/` prefix and delete their own files

#### A2. Modify `lib/persistSwing.ts`

- Change `.insert(row)` → `.insert(row).select('id').single()`
- Change return type from `Promise<void>` to `Promise<string | null>`
- Return `data.id` on success, `null` on error
- Still call `incrementLocalSwingCount()` regardless

#### A3. Create `lib/uploadSwingVideo.ts`

New file with single export:

```typescript
export async function uploadSwingVideo(swingId: string, videoPath: string): Promise<void>
```

Logic:
1. `const userId = await getUserId()` — if null (anonymous), return early
2. `const storagePath = `${userId}/${swingId}.mov``
3. `const response = await fetch(videoPath)` then `const blob = await response.blob()` — DEVICE-TEST REQUIRED (RN fetch from file:// URI)
4. `supabase.storage.from('swing-videos').upload(storagePath, blob, { contentType: 'video/quicktime', upsert: false })`
5. On success: `supabase.from('swings').update({ video_storage_path: storagePath, video_uploaded_at: new Date().toISOString() }).eq('id', swingId)`
6. On any error: `console.error(...)` only — no throw, no retry, no alert

**Fallback**: If `fetch(videoPath).blob()` fails on device, switch to `expo-file-system` `uploadAsync` — add `expo-file-system` to package.json and use its direct upload. DEVICE-TEST REQUIRED.

#### A4. Modify `app/(tabs)/record.tsx`

- Add `import { uploadSwingVideo } from '../../lib/uploadSwingVideo'`
- Add ref: `const swingIdPromiseRef = useRef<Promise<string | null> | null>(null)`
- In `finalizeCapture()` line 239, change:
  ```
  // OLD:
  persistSwing(frames, analysis, classification).catch(() => {});

  // NEW:
  swingIdPromiseRef.current = persistSwing(frames, analysis, classification).catch(() => null);
  ```
- In `beginRecording()` `onRecordingFinished` callback (line 258), add upload trigger:
  ```
  onRecordingFinished: (video) => {
    videoUriRef.current = video.path;
    // Background upload — never blocks navigation
    swingIdPromiseRef.current?.then((swingId) => {
      if (swingId) uploadSwingVideo(swingId, video.path).catch(() => {});
    });
    tryNavigate();
  },
  ```

**Why this works**: `stopRecording()` is called in `finalizeCapture()` at line 198, then `persistSwing` is called synchronously after at line 239, setting `swingIdPromiseRef.current`. The `onRecordingFinished` callback fires asynchronously after native video finalization (always after the ref is set). Upload triggers when both the swing ID resolves AND the video file is ready. Never blocks `tryNavigate()`.

#### A5. Modify `lib/supabase.ts` — deleteAccount

- Before deleting swings rows, add storage cleanup:
  ```
  const { data: files } = await supabase.storage.from('swing-videos').list(userId);
  if (files?.length) {
    await supabase.storage.from('swing-videos').remove(files.map(f => `${userId}/${f.name}`));
  }
  ```
- Place this before the existing swings delete (line 36)

#### A6. Update `lib/persistSwing.ts` APP_VERSION

- Change `const APP_VERSION = '1.2.0'` → `'1.3.0'`

### Phase B: 120fps Capture (implement after Phase A is verified)

#### B1. Modify `app/(tabs)/record.tsx` — format selection

- Add import: `useCameraFormat` from `react-native-vision-camera`
- After `device` is resolved (line 352), add format selection:
  ```typescript
  const format = useCameraFormat(device, [
    { fps: 120, videoResolution: { width: 1280, height: 720 } },
    { fps: 60 },
  ]);
  const targetFps = Math.min(format?.maxFps ?? 30, 120);
  const skipInterval = targetFps >= 120 ? 4 : targetFps >= 60 ? 2 : 1;
  ```
- Pass to Camera: `format={format}`, `fps={targetFps}`

#### B2. Modify `app/(tabs)/record.tsx` — frame skipping in worklet

- Add shared value: `const frameCounter = useSharedValue(0)`
- Reset in `beginRecording()`: `frameCounter.value = 0`
- Modify `frameProcessor` (line 372):
  ```typescript
  const frameProcessor = useFrameProcessor(
    (frame) => {
      'worklet';
      frameCounter.value = frameCounter.value + 1;
      if (frameCounter.value % skipInterval !== 0) return;
      const landmarks = honeyPoseDetect(frame);
      if (Array.isArray(landmarks) && landmarks.length > 0) {
        appendPoseFrame(landmarks, frame.timestamp, frame.width, frame.height);
      }
    },
    [appendPoseFrame, skipInterval, frameCounter]
  );
  ```

Frame skipping happens BEFORE `honeyPoseDetect` — saves CPU. Skeleton overlay updates at reduced rate (~30fps). Acceptable per requirements.

#### B3. Modify `app/(tabs)/record.tsx` — recording codec

- In `beginRecording()` `startRecording` call (line 257), add `videoCodec: 'h265'`:
  ```typescript
  cameraRef.current?.startRecording({
    videoCodec: 'h265',
    onRecordingFinished: (video) => { ... },
    onRecordingError: (e) => console.error('REC ERR:', e),
  });
  ```
- DEVICE-TEST REQUIRED: If h265 causes recording errors on some devices, remove it (fall back to default h264).

#### B4. Metadata — pass actual fps to PoseSequence

- In `finalizeCapture()`, when constructing `sequence` (line 219), add `fps` to metadata:
  ```typescript
  metadata: {
    fps: targetFps,
    durationMs: ...
  }
  ```
- This is informational only — no analysis logic reads `fps`.

## 9. Kill Criteria

- If 120fps causes preview instability, dropped frames, crashes, or thermal issues → **revert all Phase B changes, ship Phase A only**
- If `fetch(videoPath).blob()` fails in RN runtime → switch to `expo-file-system` uploadAsync
- If h265 causes recording errors → remove `videoCodec` option (default to h264)
- If format selection causes Camera to not initialize → remove `format` and `fps` props
