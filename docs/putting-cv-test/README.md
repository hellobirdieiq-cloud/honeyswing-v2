# Putting CV go/no-go artifacts

Artifacts in this directory are **manually copied** from device share-sheet
exports produced by the dev-only harness at `app/dev/putting-tracker-test.tsx`
(reachable only via `router.push('/dev/putting-tracker-test')` — no in-app
entry point). Nothing here is generated by a script or build step.

Expected files per tested swing:

- `<swingId>.json` — per-frame ball/head detections on the 8.33ms grid, gate
  metrics, and the harness's EXTERNAL ASSUMPTION constants (the native
  detector's constants are documented in
  `ios/honeyswing/HoneyPuttingTrackerPlugin.swift`).
- `<swingId>-overlay.mov` — the clip at analysis resolution with per-frame
  markers: ball = yellow dot, head = cyan dot, hollow ring = detection missing
  (drawn at last-known position), cyan rectangle = head search ROI.

## Gate targets (Phase 1 go/no-go)

| Metric | Target |
| --- | --- |
| Head-found rate during the stroke window | > 95% |
| Ball-found rate during the rest window | > 95% |
| Rest-ball jitter (stddev, full-res px) | < 2 px |

All detection thresholds are uncalibrated EXTERNAL ASSUMPTIONS until the
Phase 0 fixture corpus (10–20 varied putts + 2–3 decoys) exists. Phases 2–4
(putting phase detection, tempo bands, mode flag / Putt tab) are gated on
these targets passing and stay unbuilt until explicitly approved.
